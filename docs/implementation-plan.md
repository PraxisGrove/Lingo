# Lingo 实施路线

实施采用可独立验收的垂直切片。每一阶段都从真实入口走到可观察结果，不先铺设无人调用的抽象层。

## 阶段 0：产品化基础

- 更新包名、manifest、图标占位和 README。
- 加入标准 AGPL-3.0 文本、CLA 与贡献说明，法律文本发布前审阅。
- 建立隐私政策、权限说明和安全报告流程。
- 将现有 settings 模型升级为带 schema 版本的最小偏好模型。

验证：`pnpm compile`、`pnpm test`、`pnpm check`、Chrome/Firefox 构建均通过。

## 阶段 1：第一条端到端翻译

- 建立 Page Translation 和 Translation Orchestrator 两个 Interface。
- 内容脚本识别静态测试页的普通段落并插入译文。
- 后台通过长连接接收翻译单元，使用内存 Provider Adapter 返回结果。
- popup 支持翻译、双语对照和显示原文。

验证：一个静态文章页可以翻译并无损恢复；页面和内容脚本均无法读取测试凭据。

## 阶段 2：真实服务与首次配置

- 实现服务配置档案、凭据存储和连接测试。
- 先完成 OpenAI-compatible Adapter，再依次加入 DeepL、Google Cloud 和 Azure。
- 完成目标语言、源语言自动/手动选择与首次配置流程。
- 增加厂商契约 fixture 和错误分类。

验证：四类服务通过同一 Orchestrator Interface 工作，UI 不包含厂商分支逻辑。

## 阶段 3：可靠的页面翻译

- 实现三层内容分类和受保护内容排除。
- 保留 inline 标记、链接与强调格式。
- 支持双语对照、仅显示译文和显示原文。
- 增加 MutationObserver、IntersectionObserver 和页面 revision。
- 实现视口优先与立即翻译到底部。

验证：静态文章、无限滚动、SPA 和 iframe fixture 不重复计费，停止后无 Lingo 残留 DOM。

## 阶段 4：编排、缓存与恢复

- 实现 IndexedDB 缓存、容量管理和失效规则。
- 实现批量、并发、超时、退避、取消和局部失败。
- 完成暂停原因、失败段落重试和显式备用服务链。
- 验证后台服务工作线程重启后的重连行为。

验证：认证失败、限流、部分响应、断连和取消都有确定结果，原文始终可恢复。

## 阶段 5：自动翻译与站点规则

- 实现全局、站点和当前页面偏好解析。
- 支持站点三态策略、源语言策略和默认自动站点。
- 建立声明式规则 schema、内置规则和用户规则编辑/导入导出。
- 建立社区规则的签名、静态分发、last-known-good 和关闭更新能力。

验证：规则优先级、签名失败、离线启动和用户覆盖均有自动化测试。

## 阶段 6：AI 质量能力

- 实现有限相邻上下文和稳定段落编号协议。
- 实现术语表、翻译指令、内置模板及版本化缓存失效。
- 支持厂商原生 glossary；其他厂商通过请求约束实现，不做译后字符串替换。
- 增加结构化输出修复与逐段结果校验。

验证：混合语言、缺失段落、顺序错乱、术语冲突和长文本拆分都有固定 fixture。

## 阶段 7：完整首版交互

- 完成 popup 当前页控制中心和 options 管理界面。
- 加入快捷键、右键菜单和默认关闭的悬浮球。
- 展示缓存占用、权限、数据流向和脱敏诊断导出。
- 完成无服务、无权限、余额不足和不支持页面的空状态。

验证：键盘可达、屏幕阅读器标签、浅色/深色主题和窄弹窗均通过检查。

## 阶段 8：跨浏览器发布

- 在 Chrome、Edge 和 Firefox 执行统一验收套件。
- 审查 manifest 权限、远程规则、CSP、可复现构建和商店披露。
- 对长页面和动态页面做 CPU、内存、DOM 增长和请求数量基准。
- 生成商店包、隐私说明、权限说明和发布清单。

验证：三个目标浏览器满足产品方案的首版成功标准，CI 产物可复现。

## 后续路线

按验证结果再依次评估：Lingo 托管模型、开源同步服务与官方托管实例、账户与支付、企业受管策略、输入框翻译、PDF/EPUB、视频字幕、图片翻译、侧边栏、Safari 和移动端。

后续能力不得改变 Community 全功能开源、BYOK 完整可用和托管服务可自建的承诺。
